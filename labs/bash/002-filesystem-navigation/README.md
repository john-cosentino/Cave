# Lab 002: Filesystem Navigation and Inspection

## Objective

Practice navigating the Linux filesystem, inspecting files and directories, and understanding basic file metadata.

This lab builds comfort with moving around a system without making changes.

## Commands Practiced

- cd
- ls

- stat
- file

- tail

- less

## Questions to Answer

- What directory did the lab start in?
- What files and directories exist in the current lab folder?
- What does `tree` show that `ls` does not?
- What files can `find` locate under the lab directory?
- Which files take up the most space?
- What does `stat` show about a file?
- What does `file` identify about README.md and run.sh?
- When would I use `head` instead of `cat`?
- When would I use `tail`?
- Why is `less` safer than dumping a large file directly to the terminal?
